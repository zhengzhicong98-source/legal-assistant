# data/raw — 原始法律文本存放区

## 你只需要做一件事

把从**官方来源**下载的法律原文放到这个目录，切片器会自动识别并处理。

## 支持的格式

- `*.txt`（UTF-8，推荐）
- `*.md`
- `*.docx`（脚本会尝试读取纯文本，样式忽略）

## 命名建议

```
data/raw/
  civil-code.txt              # 民法典（骨干，1260 条）
  labor-contract-law.txt      # 中华人民共和国劳动合同法
  consumer-protection-law.txt # 消费者权益保护法
  e-commerce-law.txt          # 电子商务法
  criminal-law-excerpt.txt    # 刑法（节选，涉及大学生常见罪名）
  marriage-family-code.txt    # 民法典·婚姻家庭编（可与 civil-code 合并）
  ...
```

## 官方来源（合法可下载）

1. **国家法律法规数据库** — https://flk.npc.gov.cn/
   人大常委会官网，法律现行有效文本，纯文本可复制。
2. **中国政府网法律法规库** — http://www.gov.cn/zhengce/xxgk/
3. **中国裁判文书网** — 只作为案例参考，不入库为条文
4. **司法部普法办** — https://www.legalinfo.gov.cn/

## 文件应长什么样

切片器认识以下**结构标记**（任选其一）：

**方式 A：官方原文（推荐，几乎所有官方下载文本都是这个格式）**

```
第一条  为了保护民事主体的合法权益，……

第二条  民法调整平等主体的自然人、法人和非法人组织之间……

第三条  ……
```

只要每条以 `第XXX条` 开头，脚本会自动识别为一条 knowledge。

**方式 B：带章节标题（更好，会附加到 source 字段）**

```
第一编 总则
第一章 基本规定

第一条  为了保护民事主体的合法权益，……

第二条  ……

第二章 自然人
第一节 民事权利能力和民事行为能力

第十三条  ……
```

章节名会被自动带入 `source` 字段（如"《民法典》第一编·第一章·第一条"），提升检索时的可解释性。

## 版权与合规

- 中华人民共和国法律法规**不受著作权保护**（《著作权法》第五条第一项）
- 可以自由复制、传播、入库
- 但**司法解释、案例、学术论文**受著作权保护，请勿放入本目录
