# 需求文档

## 1. 应用概述

### 1.1 应用名称
法律助手

### 1.2 应用描述
一款面向大学生的法律知识搜索AI智能体微信小程序,以「学长陪伴」式交互帮助用户快速查询租房、求职等场景下的法律知识,提供合同审查、知识检索、文书生成、维权导航和证据采集等核心功能。

## 2. 用户与使用场景

### 2.1 目标用户
大学生及需要法律知识帮助的人群

### 2.2 核心使用场景
  - 租房时遇到合同纠纷,需要识别霸王条款
  - 求职时需要了解劳动合同相关法律知识
  - 需要生成法律文书进行维权
  - 需要查询维权渠道和办事流程
  - 需要采集和整理维权证据

## 3. 页面结构与功能说明

### 3.1 页面结构
```
├── 首页
├── 合同审查页
├── 知识检索页
├── 文书生成页
├── 维权导航页
│   └── 机构详情页
├── 证据采集页
├── 计算器工具页
└── 知识库管理页
```

### 3.2 首页
  - 展示五个核心功能入口:合同审查、知识检索、文书生成、维权导航、证据采集
  - 展示计算器工具入口
  - 自动识别用户IP位置,在页面顶部或合适位置显示当前所在城市

### 3.3 合同审查页
  - 支持用户一次性上传多个文件(租房合同、物业规约、设施交接清单等),格式支持照片或PDF文件
  - AI自动识别合同中的霸王条款,如变相扣除押金、无故解除合同不赔偿等
  - AI具备跨文档关联分析能力,识别多份文件之间的矛盾或风险点
  - 展示识别结果,标注风险条款并给出法律依据
  - 每个风险条款下方提供「学长翻译官」功能,用口语化方式解释法律条文含义
  - 每个风险条款下方展示引用的法律条文原文及其来源,格式为:来源《XX法》第XX条
  - 为合同打分(0-100分),若得分低于60分,弹窗大红字告警:「学长建议:这份合同极度危险,千万别签!」
  - 提供「隐私一键涂抹」功能,用户可对合同中的姓名、身份证号、手机号等敏感信息进行涂抹处理
  - 提供「一键生成反驳话术」功能,针对识别出的霸王条款,AI生成可直接发给房东或HR的反驳话术模版

### 3.4 知识检索页
  - 提供输入框,用户输入法律问题
  - 支持百度AI搜索功能,用户可选择使用AI搜索获取更全面的法律信息
  - AI提供场景式回复,包含:法律依据、人话翻译、话术模版、投诉渠道
  - 人话翻译以口语化方式解释法律条文,例如:「意思就是这房租你其实不用交」
  - 每个回答下方必须列出引用的法律条文原文及其来源,格式为:来源《XX法》第XX条
  - 支持多轮对话式咨询,系统自动记录对话上下文,能够理解用户的指代关系
  - 所有AI回复下方固定显示免责声明:「本回复由AI生成,仅供参考,不构成正式法律建议。若情况紧急请咨询专业律师。」

### 3.5 文书生成页
  - 提供法律文书模版选择,包括:解除劳动合同通知书、欠款催收函等
  - 用户输入基本信息后自动生成文书
  - 支持文书预览
  - 生成文书后提供「模拟对方回复」功能,AI模拟对方可能的狡辩并提供应对话术
  - 支持一键转换为长图或PDF格式,方便微信发送或打印

### 3.6 维权导航页
  - 展示全国各地劳动仲裁委、消协、法律援助中心的联系方式
  - 展示办事流程说明
  - 支持按地区筛选
  - 根据用户IP位置自动定位到当前城市的维权机构
  - 每个机构条目支持点击查看详情
  - 提供「投诉话术生成器」功能,根据用户情况生成60秒投诉话术,帮助用户在拨打12315或12348时清晰表达诉求

### 3.7 机构详情页
  - 展示机构的详细信息:名称、地址、联系电话、办公时间、办事流程等
  - 提供地图定位功能,显示机构在地图上的位置
  - 支持一键导航到该机构

### 3.8 证据采集页
  - 针对不同维权场景(如房东不退押金、劳动纠纷等)提供「证据清单自检」功能
  - 引导用户勾选已有证据类型:房屋照片、转账记录、聊天记录截屏、录音文件等
  - 根据勾选结果,AI提示用户还需补充哪些证据,并给出采集建议

### 3.9 计算器工具页
  - 提供病假工资计算器
  - 提供租房违约金计算器
  - 用户输入相关参数后自动计算结果

### 3.10 知识库管理页
  - 提供批量导入功能入口
  - 支持用户粘贴多条法律条文内容
  - 法律条文按行分隔,每行为一条独立条文
  - 点击导入后,系统自动将所有条文进行向量化处理并入库
  - 显示导入进度和结果反馈

## 4. 业务规则与逻辑

### 4.1 IP位置识别规则
  - 小程序启动时自动获取用户IP地址
  - 根据IP地址解析用户所在城市
  - 在首页顶部或合适位置显示当前城市信息
  - 在维权导航页根据识别的城市自动筛选并优先展示本地维权机构

### 4.2 合同审查规则
  - 支持上传图片格式:JPG、PNG、PDF
  - 支持一次性上传多个文件,最多支持10个文件
  - AI识别后标注风险等级:高风险、中风险、低风险
  - 每个风险条款需关联具体法律条文,并在下方展示法律条文原文及来源
  - AI需具备跨文档关联分析能力,识别多份文件之间的矛盾点
  - 合同评分规则:根据风险条款数量和风险等级综合计算,满分100分
  - 评分低于60分时弹窗告警
  - 一键生成反驳话术功能需针对每个霸王条款生成对应的反驳话术,话术需包含法律依据和建议表达方式

### 4.3 知识检索规则
  - AI回复需包含四部分:法律依据、人话翻译、话术模版、投诉渠道
  - 人话翻译需使用口语化表达,避免法律术语
  - 每个回答下方必须列出引用的法律条文原文及其来源
  - 系统需记录对话上下文,支持多轮对话中的指代理解
  - 当数据库中没有相关法律依据时,AI必须礼貌告知无法回答,严禁编造法律条文
  - 涉及复杂诉讼时需提醒用户咨询线下专业律师
  - 所有AI回复下方必须固定显示免责声明
  - 百度AI搜索功能作为可选项,用户可根据需要选择是否使用

### 4.4 文书生成规则
  - 用户填写必填信息后方可生成文书
  - 生成的文书需符合法律规范
  - 模拟对方回复功能需基于常见狡辩话术生成应对建议
  - 长图格式需适配微信聊天窗口宽度

### 4.5 维权导航规则
  - 联系方式和办事流程需定期更新
  - 支持用户按所在地区快速定位相关机构
  - 根据IP位置识别结果,自动定位到用户当前城市的维权机构
  - 投诉话术生成器需根据用户输入的维权场景和关键信息生成结构化话术
  - 点击机构条目后跳转至机构详情页

### 4.6 机构详情规则
  - 地图定位功能需调用微信小程序地图组件
  - 地图需准确标注机构位置
  - 一键导航功能需调用微信小程序导航能力,支持跳转至第三方地图应用

### 4.7 证据采集规则
  - 证据清单需根据不同维权场景动态生成
  - AI需根据用户已勾选的证据类型,智能提示缺失的关键证据
  - 提供证据采集的具体操作建议

### 4.8 计算器规则
  - 病假工资计算器:根据用户输入的基本工资、病假天数等参数计算病假工资
  - 租房违约金计算器:根据用户输入的租金、违约条款等参数计算违约金

### 4.9 知识库管理规则
  - 批量导入时,系统按行解析粘贴内容,每行视为一条独立法律条文
  - 空行自动过滤,不计入导入条目
  - 导入过程中对每条条文进行向量化处理
  - 向量化完成后自动入库
  - 导入完成后显示成功导入的条文数量

### 4.10 隐私保护逻辑
  - 用户上传的合同文件在AI分析完毕后立即销毁
  - 所有上传文件不用于模型训练
  - 在合同审查页明确提示:「所有图片分析完毕后立即销毁,不用于模型训练」
  - 提供隐私一键涂抹功能,用户可主动对敏感信息进行脱敏处理

### 4.11 免责声明逻辑
  - 所有AI生成的回复下方必须固定显示:「本回复由AI生成,仅供参考,不构成正式法律建议。若情况紧急请咨询专业律师。」
  - 免责声明需在所有涉及AI输出的页面统一展示

### 4.12 信源回溯规则
  - 所有AI回答必须在下方列出引用的法律条文原文及其来源
  - 来源格式统一为:来源《XX法》第XX条
  - 若引用多条法律条文,需逐条列出

### 4.13 多轮对话上下文规则
  - 系统需记录用户的历史对话内容
  - 当用户使用指代词(如「那」「这个」「它」等)时,系统需根据上下文准确理解指代对象
  - 对话上下文保留时长为当前会话期间

### 4.14 防幻觉机制
  - AI在回答问题时,必须严格基于知识库中的法律条文
  - 当数据库中没有相关法律依据时,AI必须礼貌告知:「抱歉,暂时无法找到相关法律依据,建议您咨询专业律师获取准确解答」
  - 严禁AI编造或虚构任何法律条文内容

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户上传的文件格式不支持 | 提示用户上传支持的格式 |
| 用户一次性上传超过10个文件 | 提示用户最多支持10个文件 |
| AI无法识别合同内容 | 提示用户重新上传清晰的文件 |
| 用户输入的问题超出法律咨询范围 | 提示用户该问题不在服务范围内 |
| 文书生成时用户未填写必填信息 | 提示用户补充必填信息 |
| 计算器输入参数不合法 | 提示用户输入有效参数 |
| 投诉话术生成器缺少关键信息 | 提示用户补充必要信息 |
| 证据清单自检时用户未勾选任何证据 | 提示用户至少勾选一项已有证据 |
| 一键生成反驳话术时未识别到霸王条款 | 提示用户当前合同未识别到需要反驳的条款 |
| 批量导入时粘贴内容为空 | 提示用户粘贴法律条文内容 |
| 批量导入时向量化处理失败 | 提示用户导入失败,请重试 |
| 地图定位失败或无法获取位置信息 | 提示用户检查定位权限或网络连接 |
| 导航功能调用失败 | 提示用户手动复制地址进行导航 |
| 百度AI搜索功能调用失败 | 提示用户搜索服务暂时不可用,请稍后重试 |
| 数据库中没有相关法律依据 | 礼貌告知用户暂时无法找到相关法律依据,建议咨询专业律师 |
| 用户使用指代词但上下文不明确 | 提示用户具体说明指代内容 |
| IP位置识别失败 | 默认显示全国范围的维权机构,用户可手动选择地区 |

## 6. 验收标准

1. 小程序启动时能够自动识别用户IP位置并在首页显示当前城市
2. 维权导航页能够根据IP位置自动定位到当前城市的维权机构
3. 用户能够成功上传单个或多个合同文件并获得AI识别结果
4. 合同审查结果中每个风险条款都包含法律依据、人话翻译和法律条文原文及来源
5. 合同评分功能正常,低于60分时能够弹窗告警
6. 隐私一键涂抹功能可正常使用
7. 一键生成反驳话术功能可针对识别出的霸王条款生成对应话术
8. 用户输入法律问题后能够获得包含法律依据、人话翻译、话术模版、投诉渠道的回复
9. 知识检索页的所有回答下方都列出引用的法律条文原文及来源
10. 知识检索页支持多轮对话,系统能够理解用户的指代关系
11. 当数据库中没有相关法律依据时,AI能够礼貌告知无法回答,不编造法律条文
12. 知识检索页的百度AI搜索功能可正常使用
13. 所有AI回复下方显示免责声明
14. 用户能够选择文书模版并生成符合规范的法律文书
15. 文书生成后能够查看模拟对方回复和应对建议
16. 文书能够一键转换为长图或PDF格式
17. 用户能够查询到全国各地维权机构的联系方式和办事流程
18. 维权导航页中点击机构条目能够跳转至机构详情页
19. 机构详情页能够正常显示机构信息和地图定位
20. 机构详情页的一键导航功能可正常使用
21. 投诉话术生成器能够根据用户情况生成结构化话术
22. 证据采集页能够根据维权场景提供证据清单自检功能
23. 用户能够使用计算器工具计算病假工资和租房违约金
24. 知识库管理页能够支持批量导入法律条文,按行分隔并自动向量化入库
25. 批量导入完成后能够显示成功导入的条文数量
26. 所有功能页面加载正常,交互流畅
